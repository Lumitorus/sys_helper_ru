# Изменение версии виртуальной аппаратуры в OVA для ESXi 5.0

Эта инструкция поможет изменить версию виртуальной аппаратуры в OVA-файле для совместимости с ESXi 5.0, если исходная версия не поддерживается.

## Шаг 1: Распаковка OVA-файла

1. Скачайте и установите **7-Zip** или **WinRAR**, если они еще не установлены.
2. Щелкните правой кнопкой мыши на файле OVA и выберите «Распаковать здесь» (или аналогичную команду).
3. В результате появится несколько файлов, включая `.ovf` и `.vmdk`.

## Шаг 2: Изменение версии виртуальной аппаратуры в OVF-файле

1. Найдите распакованный файл с расширением `.ovf`.
2. Откройте файл в текстовом редакторе, например, **Notepad++**.
3. Найдите строку, содержащую `<vssd:VirtualSystemType>`. Она может выглядеть так:
   ```xml
   <vssd:VirtualSystemType>vmx-11</vssd:VirtualSystemType>
   ```
4. Измените значение `vmx-11` на `vmx-08` или `vmx-09`, которые совместимы с ESXi 5.0:
   ```xml
   <vssd:VirtualSystemType>vmx-08</vssd:VirtualSystemType>
   ```

5. Сохраните изменения в файле `.ovf`.

## Шаг 3: Обновление файла манифеста (.mf)

Если в распакованных файлах присутствует файл с расширением `.mf` (например, `yourfile.mf`), его необходимо обновить, чтобы избежать ошибки целостности при импорте в ESXi.

1. Откройте **PowerShell** в папке, где находятся файлы.
2. Выполните следующую команду для вычисления новой SHA1-контрольной суммы для вашего `.ovf` файла:
   ```powershell
   Get-FileHash -Algorithm SHA1 "путь_к_вашему_файлу.ovf"
   ```
3. Скопируйте полученную контрольную сумму.
4. Откройте `.mf` файл в текстовом редакторе.
5. Найдите строку, начинающуюся с `SHA1(yourfile.ovf)=` и замените старую контрольную сумму на новую.

6. Сохраните файл `.mf`.

## Шаг 4: Пересборка OVA-файла

1. Снова выделите все файлы: `.ovf`, `.vmdk` и обновленный `.mf` (если он есть).
2. Щелкните правой кнопкой мыши на выделенных файлах и выберите `7-Zip -> Добавить к архиву...`.
3. В диалоговом окне выберите:
   - **Формат архива**: `tar`
   - Назовите файл как `yourfile_updated.ova`
4. Нажмите **OK** для создания нового OVA-файла.

## Шаг 5: Импорт обновленного OVA в ESXi 5.0

Теперь новый OVA-файл готов к импорту в ESXi 5.0. Он должен быть совместим с более старой версией виртуальной аппаратуры и пройти проверку целостности.

---

**Примечание**: Данная инструкция предназначена для пользователей Windows. Для Linux или macOS процесс будет аналогичным, с возможностью использования командной строки для распаковки и создания OVA.
